# 앙상블 — Ensemble

여러 모델의 예측을 합쳐 단일 모델보다 높은 성능을 내는 기법.  
모델 자체를 합치는 게 아니라 **각 모델의 예측 박스(결과)를 합산**하는 것.

## 핵심 원리

서로 다른 모델은 서로 다른 오류를 범함.  
→ 합치면 오류가 상쇄되어 전체 정확도 향상.

## 이 프로젝트의 앙상블 파이프라인

```
Drive 체크포인트 A ──→ predict.py → pred_ensemble_0.json ──┐
Drive 체크포인트 B ──→ predict.py → pred_ensemble_1.json ──┤→ WBF → submission_wbf.csv → Kaggle 제출
Drive 체크포인트 C ──→ predict.py → pred_ensemble_2.json ──┘
```

**구현 위치**: `notebooks/04_ensemble_submit.ipynb`

셀 상단 변수만 바꾸면 바로 실행 가능:

```python
WBF_CHECKPOINTS = [
    "/content/drive/MyDrive/HealthEat/checkpoints/best_model_A.pt",
    "/content/drive/MyDrive/HealthEat/checkpoints/best_model_B.pt",
]
WBF_USE_EMA = [True, True]   # 각 체크포인트 EMA 사용 여부
WBF_WEIGHTS = [2, 1]         # 성능 좋은 모델에 높게
```

`WBF_USE_EMA` / `WBF_WEIGHTS` 길이가 체크포인트보다 짧으면 마지막 값으로 자동으로 채워짐.

## 단일 모델 fallback

체크포인트가 1개이면 WBF를 건너뛰고 `submission_v1.csv`를 그대로 `submission_wbf.csv`로 복사.  
노트북 구조를 바꾸지 않고 체크포인트 1개 → 단일 모델 제출로 쓸 수 있음.

## TTA와의 차이

| | 모델 앙상블 | [[concepts/TTA]] |
|---|---|---|
| 모델 수 | 여러 개 | 1개 |
| 다양성 원천 | 다른 에폭 / 다른 설정으로 학습한 체크포인트 | 입력 이미지 변형(flip, scale 등) |
| 추론 횟수 | 모델 수 × 이미지 수 | 변형 수 × 이미지 수 |
| 학습 비용 | 체크포인트 수만큼 학습 필요 | 추가 학습 불필요 |

## 후처리 방법 비교

| 방법 | 특징 | 추천 상황 |
|---|---|---|
| [[concepts/NMS]] | 겹치는 박스 중 최고 점수만 유지 | 단일 모델 |
| [[concepts/WBF]] | 겹치는 박스를 가중평균으로 합침 | 앙상블 |

## 실험 맥락

- batch 8(mAP 94) > batch 16(mAP 92) 실험 후 단일 yolov8l 모델 기준으로 운영 중
- 체크포인트가 여러 개 쌓이면 `WBF_CHECKPOINTS`에 추가하기만 하면 앙상블 적용됨
- 현재는 같은 아키텍처(yolov8l)의 다른 에폭/설정 체크포인트를 앙상블하는 전략

## 관련 구현

- `notebooks/04_ensemble_submit.ipynb` — 앙상블 전용 노트북 (Drive 로드 → WBF → Kaggle 자동 제출)
- `ensemble_wbf.py` — WBF 계산 + CSV 생성 스크립트
- `predict.py --use-ema` — 각 체크포인트 예측 실행
