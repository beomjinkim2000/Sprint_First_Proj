# 실험 전략 — v0.2 성능 개선

> 실험 결과는 `reports/experiment_log.md`에 기록 (#56 박창준)
> 모든 실험은 동일 seed(42), 동일 val set 기준으로 비교

## 1. 모델 크기 실험 (#54 유재열)

**목표**: 성능과 속도 사이 최적 모델 크기 선택

| 모델 | 파라미터 수 | 예상 학습 시간 |
|---|---|---|
| yolov8n | 3.2M | 기준 |
| yolov8s | 11.2M | 약 2배 |
| yolov8m | 25.9M | 약 4배 |

**실험 방법**
```
configs/default.yaml의 model.name을 yolov8n → yolov8s → yolov8m 순으로 변경
동일 epochs(50), 동일 lr로 각각 학습
val_mAP + Kaggle 점수 기록
```

**결정 기준**: val_mAP 향상이 학습 시간 증가를 정당화하는 선에서 선택. Colab 시간 제한 고려.

## 2. Transfer Learning 전략 실험 (#61 유재열)

**목표**: COCO pretrained 가중치를 알약 이미지에 맞게 얼마나 조정할지 결정

**배경**: YOLOv8은 COCO(자연 이미지 80 클래스)로 학습됨. 알약 이미지는 배경이 단순하고 객체가 작아 COCO와 특성이 다름.

| 전략 | 설명 | Ultralytics 설정 |
|---|---|---|
| 전체 fine-tune | backbone 포함 전부 학습 | `freeze=0` (기본값) |
| 앞 10레이어 freeze | backbone 초반만 고정, 나머지 학습 | `freeze=10` |
| backbone 전체 freeze | detection head만 학습 | `freeze=22` |

**실험 방법**
```python
# train.py에서 freeze 파라미터 추가
model.train(data="configs/pill.yaml", freeze=N, ...)
```

**결정 기준**: freeze 많을수록 학습 빠르고 과적합 적음. val_mAP로 비교 후 선택.

## 3. 하이퍼파라미터 튜닝 (#55 박창준)

**목표**: 최적 모델(#54 결과) 기준으로 성능 극대화

**탐색 범위**

| 파라미터 | 기본값 | 탐색 범위 | 우선순위 |
|---|---|---|---|
| `lr0` (초기 lr) | 0.01 | 0.001 ~ 0.05 | 높음 |
| `epochs` | 50 | 30 / 50 / 100 | 높음 |
| `batch` | 16 | 8 / 16 / 32 | 중간 |
| `imgsz` | 640 | 640 / 1280 | 낮음 (메모리 이슈 가능) |
| `warmup_epochs` | 3 | 1 / 3 / 5 | 낮음 |

**실험 순서**: lr0 → epochs → batch 순서로 하나씩 바꾸면서 진행. 한 번에 여러 개 바꾸지 말 것.

**실험 방법**
```python
model.train(
    data="configs/pill.yaml",
    lr0=0.005,      # 변경 대상만 조정
    epochs=100,
    batch=16,
)
```

## 4. TTA — Test-Time Augmentation (#58 박창준)

**목표**: 재학습 없이 추론 정확도 향상

**방법**: `model.predict(augment=True)` 한 줄로 적용. Ultralytics가 내부적으로 flip + multi-scale 예측 후 합산.

**확인할 것**
- val_mAP 향상 여부
- 추론 시간 증가 폭 (이미지당 몇 초?)
- max_detections=4 제한이 TTA 후에도 유지되는지

## 5. WBF 앙상블 (#59 박창준 — #54 완료 후)

**목표**: 두 모델 예측을 합쳐 단일 모델보다 높은 정확도

**사용 라이브러리**: `ensemble-boxes` (`pip install ensemble-boxes`)

**파이프라인**
```
이미지 → yolov8n 예측 → 박스 N개
이미지 → yolov8s/m 예측 → 박스 M개
        ↓
     WBF 합산
        ↓
     최종 박스
```

**파라미터**
- `iou_thr`: 0.5 (겹침 판단 기준)
- `skip_box_thr`: 0.4 (이 점수 미만 박스 제외)
- `weights`: [1, 1] (두 모델 동등 비중) → 성능 좋은 모델에 더 큰 비중 실험 가능

**주의**: 두 모델 모두 학습 완료돼야 진행 가능. #54 완료 후 시작.
