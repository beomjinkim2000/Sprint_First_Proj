## ✅ 완료된 실험

### 3-Phase Fine-tuning (#61 유재열 — 완료)

COCO pretrained 가중치에서 알약 도메인으로 단계적 적응. 한 번에 전체를 fine-tune하면 COCO에서 학습한 좋은 특징이 망가질 수 있어, 헤드부터 순서대로 열어가는 방식 사용.

| 페이즈 | 언프리징 범위 | 역할 |
|--------|-------------|------|
| Phase 1 | 헤드(cv3 마지막 conv2d만) | 분류 레이어만 도메인 적응 |
| Phase 2 | 헤드 전체 | detection head 전체 튜닝 |
| Phase 3 | 전체 (백본+넥 lr 낮게, 헤드 lr 높게) | 전체 미세 조정 |

- **Optimizer**: AdamW (momentum, cosine annealing, weight decay 적용)
- **결과**: mAP@50 0.17 → 0.66 (+0.49)

## 수요일 (2026-05-27) 실험

### 1. EMA — Exponential Moving Average (#80 유재열)

**목표**: 학습 중 파라미터 노이즈를 줄여 추론 안정성 향상

**방법**: 학습 파라미터 θ의 지수이동평균 θ_ema를 별도 유지. 추론은 θ_ema로 수행.

```python
# Ultralytics ModelEMA 활용 또는 직접 구현
θ_ema = decay * θ_ema + (1 - decay) * θ  # 매 스텝마다
```

- decay 기본값: 0.9999 (학습 스텝 수에 따라 warm-up)
- 추론 시 θ_ema 사용, 학습에는 영향 없음

**확인할 것**: val mAP 안정성 향상 여부, EMA 없는 v2.2 대비 Kaggle 점수

---

### 2. 클래스별 가중치 (#81 김범진)

**목표**: 클래스 불균형으로 인해 희귀 클래스 검출이 낮은 문제 해결

**방법**
```python
# 클래스별 샘플 수 분석 후 inverse-frequency 가중치 적용
weight_c = total_samples / (num_classes * samples_c)

# Ultralytics cls_pw 파라미터 활용
model.train(..., cls_pw=weight_array)
```

**실험 순서**: 클래스 분포 분석 → weight 계산 → 적용 → per-class AP 비교

---

### 3. Ultralytics NMS (#82 박창준)

**목표**: 커스텀 NMS를 Ultralytics 내장 NMS로 교체해 일관성 확보

**현재 문제**: postprocess.py의 NMS 로직이 학습 시 Ultralytics 내부 NMS와 달라 train/inference 간 불일치 가능성

```python
from ultralytics.utils.ops import non_max_suppression

preds = non_max_suppression(
    raw_preds,
    conf_thres=0.25,
    iou_thres=0.45,
    max_det=300
)
```

**확인할 것**: 교체 전후 val mAP 변화, #69 conf_threshold 분리와 연계

---

### 4. Iterative Stratification (#83 유재열+황원재)

**목표**: 멀티라벨 환경에서 클래스 분포 균형잡힌 train/val 분할

**배경**: 기존 random 80:20 split은 희귀 클래스가 val에서만 나오거나 train에만 몰릴 수 있음

```python
from iterstrat.ml_stratifiers import MultilabelStratifiedShuffleSplit

msss = MultilabelStratifiedShuffleSplit(n_splits=1, test_size=0.2, random_state=42)
```

**확인할 것**: 분할 전후 per-class 분포 비교, val mAP 안정성 변화

---

### 5. SGD 실험 (#84 황원재+박창준)

**목표**: AdamW 대비 SGD의 일반화 성능 비교

**배경**: SGD는 AdamW보다 수렴이 느리지만 sharp minimum을 피해 일반화 성능이 높은 경우가 있음

```python
optimizer = torch.optim.SGD(
    params,
    lr=0.01,
    momentum=0.937,
    weight_decay=5e-4,
    nesterov=True
)
# + cosine annealing
```

**실험 조건**: AdamW v2.2와 동일 epoch/데이터/seed, optimizer만 교체

## 백로그 실험 (순서 미정)

### TTA — Test-Time Augmentation (#58 박창준)

재학습 없이 추론 정확도 향상. `model.predict(augment=True)` 한 줄 적용.

- **선행 조건**: #82 NMS 교체 완료 후
- **확인할 것**: val mAP 향상, 추론 시간 증가, max_det=4 제한 유지 여부

---

### WBF 앙상블 (#59 박창준)

두 모델 예측을 합쳐 단일 모델보다 높은 정확도. `ensemble-boxes` 라이브러리 활용.

- **선행 조건**: #54 상위 모델 실험 완료 후
- `iou_thr`: 0.5 / `skip_box_thr`: 0.4 / `weights`: 성능 비례

---

### 모자이크 증강 (보류)

배치당 더 많은 클래스 학습 가능하나, 테스트 이미지가 단순 구성이라 v2.1 실험에서 오히려 성능 하락 확인. 일반화 성능이 필요해질 때 재검토.
