## 마일스톤 3 실험 전략

### Q1. 앙상블이 단일 강한 모델보다 높게 나왔는데, 약한 모델 제거하면 더 오르나?
**가설** V2.7.3(val 0.9506), V4-2(val 0.9616) 포함 시 WBF 박스 위치를 흐려서 하락  
**실험** val mAP 0.97↑ 모델만 선별 WBF 재실행  
**확인** 점수 오르면 모델 선별 효과 / 안 오르면 WBF 파라미터 문제

### Q2. WBF 파라미터가 최적인가?
**가설** iou=0.5는 인접 알약 병합 위험, skip=0.25는 노이즈 박스 포함  
**근거** 알약은 이미지 내 독립 객체 / 잘 학습된 모델은 진짜 알약에 confidence 0.4↑  
**실험** iou=0.55, skip=0.3 → 완료 (결과 대기 중)

### Q3. YOLOv8x val(0.972) vs test(0.934) 괴리가 과적합인가?
**가설** val loss 곡선 없어서 현재 판단 불가  
**실험** train.py에 val_box/cls/dfl loss 추가 → train vs val loss 곡선 비교 ✅ 구현 완료  
**목적** 과적합 여부 확인 후 Q4(추가 학습) 진행 여부 결정  
**변경 내용** `train.py --run-name` 옵션 추가, `metrics_{run-name}.csv`에 val_box/cls/dfl loss 저장, 노트북 시각화 3행(train vs val loss 비교) 추가

### Q4. YOLOv8x가 333 epoch에서 plateau인가?
**실험** Q3 결과 보고 결정 — 과적합이면 중단, 아니면 500 epoch 연장  
**설정** phase3 head lr=0.00001 / backbone lr=0.000001 / wd=0.0001

### Q5. YOLO11x가 YOLOv8x보다 강한가?
**실험** 동일 설정(wd=0.0001, ep350+) 학습 후 val mAP 비교

### Q6. 해상도 1024가 성능 개선되나?
**실험** img_size=1024 재학습 (batch size 축소 필요)

### Q7. AI Hub 추가 데이터가 성능을 높이나?
**가설** 학습 데이터를 늘리면 드문 클래스의 mAP가 개선되고, 다양한 촬영 환경·배경 이미지 추가로 val→test 일반화가 좋아질 것  
**조건** `TL_2_조합.zip`, `TS_2_조합.zip` 사용 금지 (경구약제조합 5000종 중 허용 데이터셋만 사용)  
**실험** AI Hub 허용 데이터셋 확인 → 추가 학습 → val mAP / Kaggle Score 비교  
**선행 조건** Q3 val loss 곡선으로 현재 모델이 data-limited 상태인지 먼저 확인 (val loss 낮은데 test 낮으면 distribution shift, val loss 높으면 data 부족)
