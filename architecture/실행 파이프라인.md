# 실행 파이프라인

## 현재 기준

2026-05-25 기준으로 predict / postprocess / submission / dataset / train / evaluate / checkpoint 관련 PR이 main에 merge됨.
이제 작업 단위는 개별 스켈레톤 구현이 아니라, main 기준 전체 예측 파이프라인 검증과 Kaggle 1회 실행 준비다.

---

## 전체 흐름

```text
configs/default.yaml
  ↓
Dataset / DataLoader
  ↓
src/engine/predict.py
  - 1 batch model forward
  - raw model output 반환
  ↓
src/engine/postprocess.py
  - raw output → Prediction list
  - cxcywh → xyxy
  - confidence filtering
  - class-aware NMS
  - max_detections 제한
  ↓
src/submission/make_submission.py
  - Prediction list → Kaggle CSV rows
  - xyxy → xywh
  ↓
outputs/submissions/submission_vX.csv
```

---

## Step 0 — 공통 유틸리티

모든 Step에서 공통으로 사용하는 유틸리티 모음.

| 파일 | 역할 | 사용 Step | 정 | 부 |
|---|---|---|---|---|
| `src/utils/config.py` | config 로드 + null 검증 | 1, 2 | 김범진(PM) | 유재열(Model) |
| `src/utils/seed.py` | 재현성 시드 고정 | 1 | 김범진(PM) | 유재열(Model) |
| `src/utils/validate.py` | 데이터셋 / 배치 형식 검증 | 1 | 김범진(PM) | 황원재(Data) |
| `src/utils/collate.py` | DataLoader collate 함수 | 1, 2 | 김범진(PM) | 황원재(Data) |
| `src/utils/bbox.py` | xyxy↔xywh 변환 | 1, 2b | 김범진(PM) | 황원재(Data) |

## Step 1 — 학습

실행 예시: `python train.py --config configs/default.yaml`

| 파일 | 역할 | 정 | 부 |
|---|---|---|---|
| `train.py` | 진입점. config 로드, seed 고정, 루프 총괄 | 김범진(PM) | 유재열(Model) |
| `configs/default.yaml` | 하이퍼파라미터 / 경로 / 클래스 설정 | 김범진(PM) | 유재열(Model) |
| `src/data/dataset.py` | data/processed/ → `(image_tensor, target_dict)` | 황원재(Data) | 김범진(PM) |
| `src/data/transforms.py` | 이미지 + bbox 동시 augmentation | 황원재(Data) | 김범진(PM) |
| `src/data/split.py` | train/val image_id 분리 | 황원재(Data) | 김범진(PM) |
| `src/models/baseline.py` | YOLOv8 기반 모델 생성 | 유재열(Model) | 박창준(Exp) |
| `src/engine/train.py` | epoch / batch 단위 학습 루프 | 유재열(Model) | 박창준(Exp) |
| `src/engine/evaluate.py` | val mAP 계산 | 유재열(Model) | 박창준(Exp) |
| `src/engine/checkpoint.py` | checkpoint 저장 / 로드 | 유재열(Model) | 김범진(PM) |

## Step 2a — 검증 평가

실행 예시: `python predict.py --config configs/default.yaml --weights outputs/checkpoints/best.pt`

| 파일 | 역할 | 정 | 부 |
|---|---|---|---|
| `predict.py` | 루트 진입점. 가중치 로드, dataloader 순회, batch 반복 | 김범진(PM) | 박창준(Exp) |
| `src/engine/predict.py` | 1 batch 예측 → raw model output 반환 | 박창준(Exp) | 유재열(Model) |
| `src/engine/postprocess.py` | raw output → Prediction list, NMS / confidence 필터링 | 박창준(Exp) | 유재열(Model) |
| `src/data/dataset.py` | val target_dict 제공 | 황원재(Data) | 김범진(PM) |
| `src/engine/evaluate.py` | 예측 결과와 정답으로 mAP 계산 | 박창준(Exp) | 유재열(Model) |
| `reports/experiment_log.md` | 버전별 실험 결과 기록 | 박창준(Exp) | 김범진(PM) |

---

## Step 2b — Kaggle 제출

`make_submission.py`가 postprocess 결과인 Prediction list를 Kaggle CSV 형식으로 변환한다.

| 파일 | 역할 | 정 | 부 |
|---|---|---|---|
| `src/engine/predict.py` | batch tensor → raw model output | 박창준(Exp) | 유재열(Model) |
| `src/engine/postprocess.py` | raw output → Prediction list | 박창준(Exp) | 유재열(Model) |
| `src/submission/make_submission.py` | Prediction list → Kaggle CSV | 김범진(PM) | 박창준(Exp) |
| `outputs/predictions/predictions.json` | 시각화용 예측 결과 저장 | 황원재(Data) | 박창준(Exp) |
| `outputs/submissions/submission_vX.csv` | Kaggle 제출 파일 | 유재열(Model) · 박창준(Exp) | 김범진(PM) |

주의:
- 내부 bbox는 `xyxy` 절대 픽셀 좌표를 유지한다.
- Kaggle 제출 CSV에서만 `xywh` 좌상단 기준으로 변환한다.
- label과 Kaggle `category_id`가 다르면 `class_mapping.json` 기준 매핑을 `make_submission.py`에 추가해야 한다.
