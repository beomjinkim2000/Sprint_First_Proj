# 테스트 전략

## 규칙

- 파일명 `test_*.py`, 함수명 `test_`로 시작 (ADR-011)
- `if __name__ == "__main__"` 패턴 금지
- assert 없이 print로만 끝나는 검증 금지
- 테스트 입력은 `MockDataset` 텐서 또는 `DummyModel` 사용 (ADR-004)
- assert 기준은 [[architecture/모듈 설계]], [[architecture/인터페이스 계약서]] 참고
- CI `pytest tests/` 실행 예정 (ADR-012)

---

## 단위 테스트 대상

| 모듈                          | 테스트 파일                 | 입력                       | 출력 assert 기준                                 |
| --------------------------- | ---------------------- | ------------------------ | -------------------------------------------- |
| `src/data/dataset.py`       | `test_dataset.py`      | 실제 데이터 경로                | [[architecture/인터페이스 계약서]] Section 1         |
| `src/models/baseline.py`    | `test_baseline.py`     | MockDataset 텐서           | [[architecture/모듈 설계]] baseline.py           |
| `src/engine/train.py`       | `test_train.py`        | DummyModel + MockDataset | [[architecture/모듈 설계]] train.py              |
| `src/engine/predict.py`     | `test_predict_mock.py` | DummyModel + MockDataset | [[architecture/모듈 설계]] predict.py            |
| `src/engine/postprocess.py` | `test_postprocess.py`  | DummyRawOutput           | [[architecture/인터페이스 계약서]] postprocess 결과 형식 |
| `src/engine/evaluate.py`    | `test_evaluate.py`     | DummyPredDict            | [[architecture/모듈 설계]] evaluate.py           |

## 테스트 제외

| 대상                                  | 이유                             |
| ----------------------------------- | ------------------------------ |
| 루트 `train.py`, `predict.py`         | CLI 진입점. Colab에서 실제 가중치로 직접 실행 |

| `src/submission/make_submission.py` | 테스트 제외                         |
