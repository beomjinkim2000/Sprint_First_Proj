# 테스트 전략

## 규칙

- 파일명은 `test_*.py`
- 테스트 함수명은 `test_`로 시작
- `if __name__ == "__main__"` 패턴 금지
- assert 없이 print로만 끝나는 검증 금지
- 테스트 입력은 `MockDataset`, `DummyModel`, `make_dummy_raw_output`, `make_dummy_pred_dict` 사용
- assert 기준은 [[architecture/모듈 설계]], [[architecture/인터페이스 계약서]] 참고
- CI 기준 실행 명령은 `python -m pytest`

---

## 단위 테스트 대상

| 모듈 | 테스트 파일 | 입력 | 출력 assert 기준 |
|---|---|---|---|
| `src/data/dataset.py` | `test_dataset.py` | 실제/Mock 데이터 경로 | [[architecture/인터페이스 계약서]] Section 1 |
| `src/models/baseline.py` | `test_baseline.py` | MockDataset / DummyModel | 모델 생성 및 class 수 |
| `src/engine/train.py` | `test_train.py` | DummyModel + MockDataset | loss 계산 / checkpoint 흐름 |
| `src/engine/predict.py` | `test_predict_mock.py` 또는 `test_predict.py` | DummyModel + MockDataset | raw output shape |
| `src/engine/postprocess.py` | `test_postprocess.py` | DummyRawOutput / DummyPredDict | Prediction list 형식, NMS, threshold, max_detections |
| `src/engine/evaluate.py` | `test_evaluate.py` | DummyPredDict + target_dict | metric dict 형식 |
| `src/submission/make_submission.py` | `test_make_submission.py` | Prediction list | CSV columns, annotation_id, xyxy→xywh, 빈 prediction |
| `src/utils/bbox.py` | `test_bbox.py` | bbox tensor | xyxy↔xywh 변환 정확성 |
| `src/utils/validate.py` | `test_validate.py` | MockDataset | 인터페이스 위반 감지 |

## 통합 테스트 후보

### Inference → Submission 최소 연결 테스트

```text
DummyModel
  ↓
predict_batch()
  ↓
postprocess_raw_outputs()
  ↓
make_submission()
  ↓
submission.csv
```

검증 항목:
- batch raw output이 Prediction list로 변환되는지
- Prediction list가 CSV로 저장되는지
- CSV 컬럼 순서가 Kaggle 제출 형식과 같은지
- bbox가 내부 `xyxy`에서 제출용 `xywh`로 변환되는지

추천 파일명:
- `tests/test_inference_submission_pipeline.py`

## 테스트 제외 / 수동 확인 대상

| 대상 | 이유 |
|---|---|
| 루트 `train.py` | CLI 진입점. 실제 config / 데이터 / 가중치와 함께 Colab에서 확인 |
| 루트 `predict.py` | CLI 진입점. checkpoint와 test image 경로가 필요 |
| Kaggle 실제 제출 | 외부 서비스 의존. submission.csv 생성까지만 자동 테스트 권장 |
| 시각화 노트북 | 사람이 이미지 품질을 직접 확인해야 함 |

---

## 현재 우선순위

1. main 최신화 후 `python -m pytest` 전체 실행
2. 실패하는 테스트가 있으면 최신 인터페이스 기준으로 수정
3. `tests/test_inference_submission_pipeline.py` 추가
4. 실제 checkpoint 기준 submission.csv 1회 생성
