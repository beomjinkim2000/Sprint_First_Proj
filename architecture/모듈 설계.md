# 모듈 설계

## 전체 파이프라인

```text
Data(raw / processed)
  └─ src/data/dataset.py
       │  image tensor + target_dict
       ▼
Model
  └─ src/models/baseline.py
       │
       ▼
Train / Evaluate
  ├─ src/engine/train.py
  ├─ src/engine/evaluate.py
  └─ src/engine/checkpoint.py
       │
       ▼
Predict  (1 batch forward)
  └─ src/engine/predict.py
       │  raw model output
       ▼
Postprocess
  └─ src/engine/postprocess.py
       │  List[Prediction]
       ▼
Submission CSV
  └─ src/submission/make_submission.py
```

---

## 모듈별 책임 & 입출력

### `src/data/dataset.py`
- **책임**: 파일을 읽어서 Tensor + target_dict 반환
- **입력**: 이미지 경로 목록, annotation JSON/CSV
- **출력**: `(image_tensor [C,H,W], target_dict)`
- **주의**: bbox는 내부 표준인 `xyxy` 절대 픽셀 좌표로 반환

### `src/data/transforms.py`
- **책임**: 이미지와 bbox를 동시에 변환
- **입력**: image, target_dict
- **출력**: 변환된 image, target_dict
- **주의**: resize / augmentation 시 bbox도 같이 변환해야 함

### `src/data/split.py`
- **책임**: train/val image_id 분리
- **입력**: 전체 image_id 리스트, val_ratio
- **출력**: `(train_ids, val_ids)`

### `src/models/baseline.py`
- **책임**: YOLOv8 기반 모델 객체 생성
- **입력**: config 또는 num_classes
- **출력**: `torch.nn.Module`
- **주의**: 클래스 수 변경은 모델 생성 단계에서 처리

### `src/engine/train.py`
- **책임**: 학습 루프
- **입력**: model, dataloader, optimizer, device 등
- **출력**: loss / metric / checkpoint 저장 흐름
- **주의**: dataset의 내부 구현이 아니라 target_dict 인터페이스에만 의존

### `src/engine/evaluate.py`
- **책임**: validation 성능 측정
- **입력**: model, dataloader, device
- **출력**: `{"mAP": float, "mAP_50": float, ...}`
- **주의**: 예측 결과는 postprocess 결과 형식과 target_dict를 기준으로 비교

### `src/engine/checkpoint.py`
- **책임**: checkpoint 저장 / 로드
- **입력**: model, optimizer, epoch, metric, path 등
- **출력**: `.pt` checkpoint 또는 로드된 상태
- **주의**: 루트 predict.py가 학습된 모델을 불러올 때 의존

### `src/engine/predict.py`
- **책임**: 1 batch 예측 → raw model output 반환
- **입력**: model, batch tensor, device
- **출력**: raw model output
- **주의**: `model.eval()`은 루트 predict.py에서 수행. engine predict.py는 model forward 단일 책임만 가진다.

### `src/engine/postprocess.py`
- **책임**: raw output을 제출/평가 가능한 Prediction list로 변환
- **입력**: raw predictions, image_ids, postprocess config
- **출력**: `List[Prediction]`
- **처리**: `cxcywh → xyxy`, confidence filtering, class-aware/class-agnostic NMS, max_detections 제한
- **주의**: 내부 bbox 형식은 `xyxy` 절대 픽셀 좌표 유지

### root `predict.py`
- **책임**: checkpoint 로드, test dataloader 순회, batch 예측 반복, submission용 prediction 저장
- **입력**: config, checkpoint, test dataset
- **출력**: `outputs/predictions/predictions.json`, `outputs/submissions/submission_vX.csv`
- **처리**: 640 기준 bbox를 원본 이미지 좌표로 scale back, bbox clamp, 비정상 크기 bbox filtering
- **주의**: validation/resize 시각화에서는 640 기준 좌표를 유지하고, Kaggle submission에서는 원본 이미지 기준 좌표를 사용한다.

### `src/submission/make_submission.py`
- **책임**: Prediction list → Kaggle 제출 CSV
- **입력**: `List[Prediction]`
- **출력**: `outputs/submissions/submission_vX.csv`
- **처리**: `xyxy → xywh`, annotation_id 생성, CSV 저장, label → category_id 매핑
- **주의**: label과 Kaggle `category_id`는 `configs/class_mapping.json` 기준으로 변환한다.

### `src/utils/bbox.py`
- **책임**: bbox 형식 변환 공통 함수
- **함수**: `xyxy_to_xywh()`, `xywh_to_xyxy()`

### `src/utils/seed.py`
- **책임**: 재현성 시드 고정
- **함수**: `set_seed(seed: int)`

### `src/utils/config.py`
- **책임**: config 로드 + null 검증
- **함수**: `load_config(path)`

### `src/utils/collate.py`
- **책임**: DataLoader collate 함수
- **함수**: `collate_fn(batch)`

### `src/utils/validate.py`
- **책임**: 데이터셋/배치 형식 검증
- **함수**: 인터페이스 계약 형식 준수 여부 검사

## 응집도 / 결합도 원칙

| 원칙 | 이 프로젝트에서 적용 |
|---|---|
| 응집도 높이기 | 각 `.py` 파일이 하나의 책임만 가짐 |
| 결합도 낮추기 | 모듈은 서로의 내부 구현이 아니라 인터페이스 형식에 의존 |
| 인터페이스로 연결 | 모듈 간 연결은 [[architecture/인터페이스 계약서]] 형식을 따름 |
| 변환 위치 고정 | 학습용 변환, 후처리용 변환, 제출용 변환을 각 책임 모듈에 둠 |
