fallback- 필터링 후 박스가 너무 작으면 제거된 후보중score가 높은 것 부터 살리는거 


1. **Soft-NMS**
    
    - 겹치는 박스를 바로 삭제하지 않고 score만 낮춤.
    - 가까이 붙은 알약이 있을 때 일반 NMS보다 덜 과격함.
    - 단, Ultralytics 기본 NMS를 대체해야 해서 구현 부담 있음.
2. **Weighted Boxes Fusion, WBF**
    
    - 비슷한 위치의 여러 박스를 하나로 평균내서 합침.
    - TTA나 여러 모델 결과를 합칠 때 특히 좋음.
    - 단일 모델 1회 예측에는 효과가 제한적일 수 있음.
3. **TTA 후처리**
    
    - 원본, 좌우반전, 상하반전 등으로 여러 번 예측하고 결과를 다시 합침.
    - missed detection 줄이는 데 도움 될 수 있음.
    - 단점은 추론 시간이 늘어남.
4. **per-class top-k / per-class threshold**
    
    - 클래스별로 너무 많이 튀는 클래스는 threshold 높이고, 잘 안 잡히는 클래스는 낮춤.
    - 클래스별 AP/오탐 분석이 있어야 의미 있음.
5. **box voting**
    
    - NMS로 선택된 박스를 주변 후보 박스들의 가중 평균으로 보정.
    - 박스 위치를 조금 더 안정화하는 기법.
    - WBF의 가벼운 버전처럼 볼 수 있음.
